## Contributing New Examples

When adding a new project link, please:

1. **Verify Quality**: Ensure the project is:
   - Well-documented
   - Actively maintained (or a completed reference implementation)
   - Demonstrates best practices
   - Open source or publicly accessible

2. **Use the Template**:
   ```markdown
   #### [Project Name]
   - **Link**: [URL]
   - **Description**: One sentence about what it does
   - **Technologies**: List main technologies/tools
   - **Key Features**:
     - Notable feature 1
     - Notable feature 2
     - Notable feature 3
   - **Why It's Notable**: What makes this worth studying
   ```

3. **Choose the Right Category**: Place under the most relevant category
4. **Keep It Current**: Mark projects as archived if they're no longer maintained
5. **No Self-Promotion Without Merit**: Only add projects that genuinely demonstrate excellence

## Project Selection Criteria

Projects should meet at least 3 of these criteria:

- [ ] Demonstrates innovative use of AI
- [ ] Shows best practices in code organization
- [ ] Has comprehensive documentation
- [ ] Includes real-world use cases
- [ ] Provides reusable patterns/templates
- [ ] Has active community engagement
- [ ] Showcases effective prompt engineering
- [ ] Implements novel approaches
- [ ] Production-ready quality
- [ ] Educational value
